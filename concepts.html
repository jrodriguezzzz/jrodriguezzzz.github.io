Key Concepts

ConceptDefinition
HTTPThe foundational protocol used by the World Wide Web to dictate how messages are formatted and transmitted. Understanding HTTP is essential because it governs every interaction between a web browser and a server when fetching web pages.
TCPA foundational networking standard that ensures data is delivered reliably and in the correct order between devices. For decades, HTTP relied entirely on TCP to guarantee that web content didn't get lost in transit, though its strict error-checking can sometimes cause delays.
UDPA faster, connectionless communication protocol that sends data without guaranteeing its delivery or order. HTTP/3 harnesses UDP (via QUIC) to drastically speed up web traffic and eliminate the bottlenecks historically caused by TCP's strict reliability checks.
LatencyThe delay in communication over a network, measuring the time it takes for a data packet to travel from its source to its destination. Reducing latency has been the driving force behind every major HTTP upgrade, as lower latency means websites load significantly faster for users.
MultiplexingThe process of sending multiple data streams over a single communications channel simultaneously. In HTTP/2, multiplexing revolutionized web performance by allowing a browser to download multiple images, scripts, and stylesheets at exactly the same time over one connection.
TLS/SSLCryptographic protocols designed to provide secure, encrypted communication over a computer network. Modern web iterations like HTTP/2 and HTTP/3 actually mandate the use of TLS (specifically under HTTPS) to ensure that user data remains private and protected from tampering.
Head-of-Line BlockingA performance issue where a line of data packets is held up by the first packet experiencing a delay. Solving this specific bottleneck, first at the application layer in HTTP/2 and then at the transport layer in HTTP/3, has been the central challenge of the modern web evolution.
