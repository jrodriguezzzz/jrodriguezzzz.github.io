References

  • Berners-Lee, T. (1991). The Original HTTP as Implemented. W3C.
    Comment: Explains the original, rudimentary design of HTTP/0.9 and its core GET method.
  • Fielding, R., et al. (1996). RFC 1945: HTTP/1.0. IETF.
    Comment: This document provides the formalization of HTTP/1.0, codifying the use of HTTP headers and status codes.
  • Fielding, R., et al. (1999). RFC 2616: HTTP/1.1. IETF.
    Comment: Details the introduction of persistent connections (keep-alive) to solve early TCP latency issues.
  • Thomson, M., et al. (2022). RFC 9113: HTTP/2. IETF.
    Comment: Details the technical specifications of HTTP/2, outlining the fundamental shift to a binary framing layer and multiplexing.
  • Bishop, M. (2022). RFC 9114: HTTP/3. IETF.
    Comment: Outlines the most recent iteration of HTTP, detailing the shift to QUIC over UDP.
  • Iyengar, J., & Thomson, M. (2021). RFC 9000: QUIC: A UDP-Based Multiplexed and Secure Transport. IETF.
    Comment: Defines the core QUIC transport protocol that replaces TCP in HTTP/3, enabling faster connection establishment.
  • Grigorik, I. (2013). High Performance Browser Networking. O'Reilly Media.
    Comment: Provides an in-depth look at how web performance is impacted by underlying network protocols like TCP and HTTP.

  • Mozilla. (n.d.). MDN Web Docs: HTTP.

  • http.dev. (n.d.). http.dev - HTTP explained.
